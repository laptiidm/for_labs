## 4. Висновки та контрольні запитання

### 4.1 Контрольні запитання

1. **Скетч** – простий начерк сторінки, призначений для швидкого показу ідеї.  
2. **Варфрейм** – структурний макет без кольорів і стилів, призначений для продумування логіки.  
3. **Мокап** – візуально оформлений макет, показує дизайн сторінки.  
4. **Прототип** – інтерактивна модель сторінки для тестування UX.  
5. **Інформаційні елементи прототипу:** тексти, таблиці, зображення, повідомлення.  
6. **Функціональні елементи:** кнопки, поля введення, чекбокси, випадаючі списки.  
7. **Навігаційні елементи:** меню, хедер, сайдбар, панель переходів.  
8. **Елементи хедера:** логотип, назва сайту, меню, пошук, кнопки авторизації.  
9. **Елементи сайдбару:** категорії, фільтри, посилання, віджети.  
10. **Елементи контентної частини:** тексти, картинки, таблиці, форми, блоки дії.  
11. **Блоки з цільовою дією:** кнопки, форми для залучення користувача.  
12. **Проєктування форм:** короткі поля, підказки, логічне розташування, мінімум кроків.  
13. **Статичний прототип:** неінтерактивний; **Динамічний:** інтерактивний, демонструє поведінку.  
14. **Порівняння скетчу, варфрейму, мокапу, прототипу:**  
    - Скетч: низька деталізація, без дизайну, без інтерактивності  
    - Варфрейм: середня деталізація, без дизайну, без інтерактивності  
    - Мокап: висока деталізація, дизайн присутній, без інтерактивності  
    - Прототип: висока деталізація, дизайн присутній, інтерактивний  
15. **Порівняння сучасних інструментів:**  
    - Balsamiq: простий для wireframe, мало інтерактивності  
    - Figma: інтерактивні прототипи, сучасний дизайн  
    - Canva: швидко робити гарний вигляд, не для UX  
    - Visual Paradigm: UML + UI, серйозний для бізнес, громіздкий
