## 🔹 1. Створення середовища (Day 1)

### 1.1 Репозиторій
Створити проєкт на GitHub.
**Рекомендована структура:**
```text

/docs     # Схеми та опис
README.md # Головний файл
```

### 1.2 README (мінімальний)
* [ ] Опис проєкту.
* [ ] Стек технологій (ASP.NET Core).
* [ ] Інструкція із запуску (заповнити пізніше).

### 1.3 (Опційно) Документація
Створити простір у **Confluence** або окрему папку `/docs` в репозиторії.

## 🔹 2. Визначення MVP (Day 1–2)
> 💡 **Ключовий етап** — не ускладнювати!

### Мінімальний функціонал:
| Роль | Можливості |
| :--- | :--- |
| **User** | Перегляд доступних слотів, бронювання слоту. |
| **Admin** | Створення нових слотів, зміна статусу (вільний / зайнятий). |

### Модель даних (проста)
```csharp
Slot {
    Id: int,
    StartTime: DateTime,
    EndTime: DateTime,
    IsBooked: bool,
    Comment: string (optional)
}
```
**Результат:** файл `requirements.md`.

---

## 🔹 3. Архітектура (Day 2)
> 🎓 **Порада:** Чіткий поділ на шари.

### Основні шари:
* **Models:** Сутності даних.
* **Repositories:** Робота з базою даних (напр. `ISlotRepository`).
* **Services:** Бізнес-логіка (напр. `SlotService`).
* **Controllers:** API ендпоінти (`SlotController`).

**Результат:** файл `architecture.md`.

---

## 🔹 4. Розбиття на задачі (Day 2)
Використовуйте **Trello** або **GitHub Issues**.

### Приклад backlog-у:
* **Setup:** Створити ASP.NET Core проєкт, налаштувати структуру папок.
* **Backend:**
    * [ ] Створити модель `Slot`.
    * [ ] Реалізувати Repository та Service.
    * [ ] Створити API: `GET /slots`, `POST /book`.
* **Frontend:** Проста сторінка зі списком та кнопкою "Забронювати".
* **Docs:** Описати API, підготувати презентацію.
